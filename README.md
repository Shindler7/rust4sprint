# Обработчик изображений с плагинами

Учебный проект: CLI-приложение для обработки изображений с поддержкой
динамически подключаемых плагинов через FFI.

## Возможности

- обработка изображений через консольное приложение `iproc`
- подключение новых плагинов без изменений основного приложения

## Структура проекта

- `image_processor` (`iproc`) — консольное приложение для обработки изображений
- отдельные крейты плагинов (libs):
    - `blur_plugin` — размытие изображения
    - `mirror_plugin` — зеркальное отражение изображения по вертикали и (или)
      горизонтали

## Установка

### Базовые требования к системе

- [Rust Toolchain](https://rust-lang.org/tools/install/) (Rust и Cargo, версия
  1.90+)
- Система контроля версий [Git](https://clck.ru/V9Qx2)

### Сборка

Все крейты объединены в одном рабочем пространстве (`workspace`), поэтому
компилируются одной командой.

```shell
git clone git@github.com:Shindler7/rust4sprint.git
cd rust4sprint
cargo build --release
```

## Быстрый старт

### Запуск iproc

При запуске `iproc` передаются параметры:

- **input** (`-i`, `--input`) — путь к исходному PNG-изображению
- **output** (`-o`, `--output`) — путь к директории сохранения результата (имя
  выходного файла дополняется случайным хешем)
- **params** (`-p`, `--params`) — путь к текстовому файлу конфигурации плагина
- **plugin-path** (`-P`, `--plugin-path`) — (опционально) путь к директории,
  где
  находится плагин (в режиме разработки по умолчанию `target/debug`, в релизном
  запуске — корневая директория проекта)
- **plugin** (`-n` `--plugin`) — название плагина (например, `mirror`)

Пример использования плагина `mirror` в режиме разработки:

```shell
cargo run --bin iproc -- -i ".cp/flowers.png" -o ".cp/mirror" -p ".cp/mirror/mirror.cfg" -n "mirror"
```

Вызов того же плагина в релизной версии:

```shell
iproc -i ".cp/flowers.png" -o ".cp/mirror" -p ".cp/mirror/mirror.cfg" -n "mirror"
```

### Справка

Как и другие консольные приложения `iproc` предоставляет справку по своим
возможностям:

```shell
iproc --help
```

### run.sh

Для удобства в корне workspace есть скрипт:

```shell
./run.sh mirror debug
```

Позволяет задать статичные пути и выбрать вариант запуска приложения: `debug`
или `release`, и требуемый плагин.

## Плагины

Все плагины экспортируют C ABI-функцию:

`process_image(width: u32, height: u32, rgba_data: *mut u8, params: *const c_char)`,
где `rgba_data` — буфер RGBA длиной `width * height * 4`.

### `mirror`

Плагин для отражения изображения по вертикали и/или горизонтали.

- **Крейт:** `mirror_plugin` (написан на C)
- **Параметры:**
    - `horizont=0|1` — отражение по горизонтали (`1` — true, `0` — false)
    - `vertical=0|1` — отражение по вертикали (`1` — true, `0` — false)

Пример:
`horizont=1,vertical=0`

> Рекомендуется передавать оба параметра явно.

### `blur`

Плагин для размытия изображения.

- **Крейт:** `blur_plugin`
- **Параметры:**
    - `radius=n` — радиус размытия (`n` > 0)
    - `iterations=n` — количество итераций (`n` > 0)

Если параметры не переданы или не распарсились:

- `radius = 1`
- `iterations = 100`

Пример:
`radius=2,iterations=20`

#### Важно

При внесении изменений в код плагина, в режиме разработки необходимо его
пересобрать:

```shell
cargo build -p blur_plugin
```
